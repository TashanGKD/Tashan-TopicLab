## P0-A：messages 持久化（L1 + L2）

### 根本原因

`sessions.py` 中 session 完全存内存，`messages`（LLM 对话历史）从不写磁盘。TTL=3600 秒后 session 被 `_cleanup()` 删除，AI 上下文彻底消失。前端 localStorage 只保存 UI 显示层，后端创建新 session 时 `messages=[]`，造成前后端状态分裂。

### 修改方案

#### 修改 1：`sessions.py` — 新增消息持久化函数

在 `sessions.py` 中新增以下函数（在 `save_profile` 之后添加）：

```python
# ── 消息历史持久化路径 ──────────────────────────────────────────────────────
def _messages_path(session: dict) -> Path:
    """返回该 session 的消息历史文件路径。"""
    user_id = session.get("user_id")
    if user_id:
        return _profiles_dir(user_id) / "messages.json"
    else:
        # 匿名用户：按 session_id 前 8 位命名
        sid = session["session_id"][:8]
        anon_dir = _profiles_dir(None)
        anon_dir.mkdir(parents=True, exist_ok=True)
        return anon_dir / f"messages-{sid}.json"


def save_messages(session: dict) -> None:
    """将 session.messages 持久化到磁盘（每次 run_block_agent 后调用）。"""
    path = _messages_path(session)
    path.parent.mkdir(parents=True, exist_ok=True)
    # 只保存 user/assistant 两种角色（剔除 tool 角色，降低体积）
    # tool 角色的 content 对 AI 续接有帮助但较大，可按需保留
    storable = [
        m for m in session.get("messages", [])
        if m.get("role") in ("user", "assistant", "tool")
    ]
    path.write_text(
        __import__("json").dumps(storable, ensure_ascii=False, indent=None),
        encoding="utf-8"
    )


def load_messages(session_id: str, user_id=None) -> list:
    """在 session 重建时尝试从磁盘恢复消息历史。"""
    import json
    # 构造候选路径
    if user_id:
        path = _profiles_dir(user_id) / "messages.json"
    else:
        sid = session_id[:8]
        path = _profiles_dir(None) / f"messages-{sid}.json"
    if path.exists():
        try:
            return json.loads(path.read_text(encoding="utf-8"))
        except Exception:
            return []
    return []
```

#### 修改 3：`profile_helper.py` — chat/blocks 接口在响应后保存 messages

在 `POST /chat/blocks` 的 `generate()` 函数中，`run_block_agent` 调用完成后触发保存：

```python
def generate():
    try:
        blocks = profile_block_agent.run_block_agent(
            req.message, session, model=req.model
        )
        # ── 新增：保存消息历史到磁盘 ──────────────────────────────────
        profile_sessions.save_messages(session)
        # ────────────────────────────────────────────────────────────
        for block in blocks:
            yield f"data: {json.dumps(block, ensure_ascii=False)}\n\n"
    except Exception as e:
        yield f"data: {json.dumps({'type': 'text', 'content': f'服务器错误: {e}'})}\n\n"
    yield "data: [DONE]\n\n"
```

> **为何在 API 层而非 agent 层调用**：`run_block_agent` 是同步函数，不感知 I/O 上下文；API 层是最合适的写入点，且此时 session.messages 已被 block_agent 完整更新。

#### 修改 4：清理时不删除消息文件（session 过期只清内存）

`_cleanup()` 中当前只 `_sessions.pop(sid, None)`（删内存），消息磁盘文件天然保留，**无需额外修改**。消息文件的清理可通过单独的定时任务（或部署层脚本）按文件 mtime 清理过期文件。

### 验证（推演）

```
场景：用户问 Q1 Q2 Q3，关闭浏览器，3小时后回来

修改前：
  → session 被 _cleanup() 删除，messages=[]
  → 前端 localStorage 显示 Q1/Q2/Q3 历史（UI层），后端 AI 不知道
  → AI 以为新用户，从头开始 ❌

修改后：
  → generate() 在每轮对话后调用 save_messages()
  → messages-{sid[:8]}.json 写入磁盘
  → 3小时后用户回来 → get_or_create() → _new_session() → load_messages() 读取磁盘
  → session.messages = [Q1对话, Q2对话, Q3对话]
  → AI 携带完整历史，知道 Q1-Q3 已回答，继续问 Q4 ✅
```

## P0-B：单次只允许一个交互 Block（L5 + L6）

### 根本原因

**L5**：`block_agent.py` 的工具调用循环 `for tc in tool_calls` 对所有 `ask_xxx` 都 append block，LLM 单次可调用多个。

**L6**：`ChatWindow.tsx` 的 `_responded` 标记在**消息级**（整条 message），点击第一个 block 后整条消息变为已响应，其余 block 永久禁用，第二个问题的答案丢失。

### 修改方案

#### 修改 5：`block_agent.py` — 强制单次只允许一个交互 Block

在 `run_block_agent` 的工具调用循环中，增加对交互型 block 的计数限制：

```python
has_interactive_ui = False
interactive_block_count = 0   # ← 新增计数器

for tc in tool_calls:
    fn_name = tc.function.name
    try:
        args = json.loads(tc.function.arguments or "{}")
    except json.JSONDecodeError:
        args = {}

    if fn_name in _INTERACTIVE_UI_TOOLS:
        # ── 新增：单次只允许一个交互 Block ────────────────────────────
        if interactive_block_count >= 1:
            # 告知 LLM 本次只能出一个问题，稍后再问
            messages.append({
                "role": "tool",
                "tool_call_id": tc.id,
                "content": (
                    "【系统限制】每轮对话只能向用户展示一个问题。"
                    "此工具调用已被忽略，请等用户回答当前问题后，再在下一轮中提问。"
                )
            })
            continue
        # ─────────────────────────────────────────────────────────────
        block = _ui_tool_to_block(fn_name, args)
        response_blocks.append(block)
        messages.append({
            "role": "tool",
            "tool_call_id": tc.id,
            "content": "已展示给用户，请等待用户回复后再继续。"
        })
        has_interactive_ui = True
        interactive_block_count += 1   # ← 新增

    elif fn_name in _DISPLAY_UI_TOOLS:
        block = _ui_tool_to_block(fn_name, args)
        response_blocks.append(block)
        messages.append({
            "role": "tool",
            "tool_call_id": tc.id,
            "content": "已展示给用户。"
        })
    else:
        # 后端工具，执行并喂回 LLM
        result = _execute_backend_tool(fn_name, args, session)
        messages.append({
            "role": "tool",
            "tool_call_id": tc.id,
            "content": result
        })

if has_interactive_ui:
    break
```

```python
META_SYSTEM_PROMPT = """

## 【强制规则】提问节奏（不可违反）

1. **每轮只能提一个问题**：在一次工具调用响应中，只能调用一次 ask_choice、ask_text 或
```

#### 修改 8：`ChatWindow.tsx` — handleBlockRespond 改为 Block 级标记

找到 `handleBlockRespond` 函数，修改标记逻辑：

```typescript
const handleBlockRespond = useCallback(
  async (msgIndex: number, responseText: string, blockId?: string) => {
    // ── 修改：改为 Block 级标记，而非消息级 ──────────────────────────
    setMessages((prev) => {
      const next = [...prev]
      const msg = next[msgIndex]
      if (msg?.role === 'assistant') {
        const respondedBlocks = new Set(msg._responded_blocks ?? [])
        if (blockId) {
          respondedBlocks.add(blockId)
        }
        // 如果所有交互型 block 都已响应，则标记整条消息为已响应
        const interactiveBlocks = (msg.blocks ?? []).filter(
          (b) => b.type === 'choice' || b.type === 'rating' || b.type === 'text_input'
        )
        const allResponded = interactiveBlocks.every((b) => {
          const id = 'id' in b ? (b as { id: string }).id : undefined
          return id ? respondedBlocks.has(id) : false
        })
        next[msgIndex] = {
          ...msg,
          _responded_blocks: respondedBlocks,
          _responded: allResponded,   // 向后兼容
        } as ChatMessage
      }
      return next
    })
    // ... 后续 handleSend 逻辑不变 ...
  },
  [sessionId, loading, selectedModel, requireCurrentUser, fetchProfile]
)
```

### 修改方案

#### 修改 11：`profile_helper.py` — 新增对话历史查询接口

```python
@router.get("/chat-history/{session_id}")
async def get_chat_history(
    session_id: str,
    auth_ctx: dict = Depends(get_current_auth_context),
):
    """返回 session 的用户可见对话历史（过滤 tool 角色消息）。"""
    uid = _get_uid(auth_ctx)
    normalized = _normalize_session_id(session_id)
    session = _get_session_for_user(normalized, auth_ctx)

    # 优先从内存 session 读取；若内存无则从磁盘读取
    messages = session.get("messages") or profile_sessions.load_messages(normalized, uid)

    # 过滤为用户可见消息（去除 tool 角色，去除 blocks=[] 的占位符）
    user_visible = []
    for m in messages:
        role = m.get("role")
        if role == "user":
            user_visible.append({"role": "user", "content": m.get("content", "")})
        elif role == "assistant":
            content = m.get("content", "")
            if content:   # 有文字内容的 assistant 消息（非工具调用中转消息）
                user_visible.append({"role": "assistant", "content": content})

    return {"messages": user_visible, "count": len(user_visible)}
```

```python
def _check_profile_completeness(profile: str) -> dict:
    """解析 profile.md 内容，返回 F1-F7 各字段的填写状态。"""
    import re

    def has_content(pattern: str) -> bool:
        """检查 profile 中某行是否有实质内容（非空白/占位符）。"""
        match = re.search(pattern, profile, re.MULTILINE)
        if not match:
            return False
        value = match.group(1).strip()
        placeholders = {"待填写", "—", "-", "", "TBD", "暂无"}
        return value not in placeholders and len(value) > 0

    f1 = has_content(r"研究阶段[：:]\s*(.+)")
    f2 = has_content(r"一级领域[：:]\s*(.+)")
    f3 = has_content(r"方法范式[：:]\s*(.+)")
    # F4：科研流程能力至少3个维度有评分（检查表格行数）
    flow_rows = re.findall(r"\|\s*(问题定义|文献整合|方案设计|实验执行|论文写作|项目管理)\s*\|[^|]+\|\s*([1-5])", profile)
    f4 = len(flow_rows) >= 3
    # F5：技术能力至少1行
    tech_rows = re.findall(r"\|\s*\w+\s*\|\s*\S+\s*\|\s*[★☆]{1,5}", profile)
    f5 = len(tech_rows) >= 1
    # F6：主要时间占用至少1条
    f6_section = re.search(r"### 3\.1.*?### 3\.2", profile, re.DOTALL)
    f6 = bool(f6_section and len(f6_section.group(0)) > 50)
    # F7：核心难点至少1条
    f7_section = re.search(r"### 3\.2.*?### 3\.3", profile, re.DOTALL)
    f7 = bool(f7_section and len(f7_section.group(0)) > 50)

    all_filled = all([f1, f2, f3, f4, f5, f6, f7])
    missing = []
    if not f1: missing.append("F1：研究阶段")
    if not f2: missing.append("F2：一级领域")
    if not f3: missing.append("F3：方法范式")
    if not f4: missing.append("F4：科研流程能力（至少3维度）")
    if not f5: missing.append("F5：技术能力（至少1条）")
    if not f6: missing.append("F6：主要时间占用（至少1条）")
    if not f7: missing.append("F7：核心难点（至少1条）")

    return {
        "all_required_filled": all_filled,
        "missing_fields": missing,
        "detail": {"F1": f1, "F2": f2, "F3": f3, "F4": f4, "F5": f5, "F6": f6, "F7": f7}
    }
```

```python
def migrate_anon_to_user(session: dict, user_id) -> None:
    """将匿名 session 的画像、消息迁移到用户目录，并更新 session.user_id。"""
    pdir = _profiles_dir(user_id)
    pdir.mkdir(parents=True, exist_ok=True)

    # 迁移 profile
    if session.get("profile"):
        target = pdir / "profile.md"

    # 迁移 messages（覆盖写到用户目录的 messages.json）

    import json
    messages_path.write_text(
        json.dumps(session.get("messages", []), ensure_ascii=False),
        encoding="utf-8"
    )

    # 更新 session.user_id
    session["user_id"] = user_id
```

```
    ▼ generate()
    ├── save_messages(session)  ← 【新增】写入 messages.json
```

## 遗留风险与已知限制

| 风险 | 说明 | 接受/规避 |
|---|---|---|
| 匿名用户清浏览器缓存 | sessionId 丢失，无法恢复（匿名模式固有限制） | 接受，引导用户登录 |
| messages.json 体积增长 | 长期用户的对话历史文件会增大 | 定期清理（部署层 cron），或限制保存最近 N 条 |
| LLM 路由仍是概率行为（L10）| LLM 偶尔不调 read_skill，直接回答 | 接受（META_SYSTEM_PROMPT 约束+熔断保护，概率极低）|
| collect-basic-info 写盘窗口期 | Q1-Q5 5问内断线仍无 profile，但 messages 有历史 | 接受（messages保障AI续接，profile 5问完成后补写）|
| messages.json 过期清理 | 磁盘文件永不过期（TTL 只在内存层） | 建议定期清理 >90天的文件 |
